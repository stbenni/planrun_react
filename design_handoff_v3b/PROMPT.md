# Промпт для Claude Code (скопируй целиком в начало сессии)

Мы делаем полный редизайн PlanRun по утверждённому дизайну v3B «Телеметрия».
Пакет дизайна лежит в `design_handoff_v3b/` (в корне репо). Прочти в этом порядке:
1. `design_handoff_v3b/README.md` — главная директива и порядок работ
2. `design_handoff_v3b/SCREENS.md` — маппинг моков на файлы нашего кода
3. `design_handoff_v3b/tokens.css` — дизайн-токены
4. Моки экранов: `design_handoff_v3b/mocks/*.jsx`
5. Открой `design_handoff_v3b/prototype.html` в браузере, чтобы видеть поведение

КРИТИЧЕСКОЕ ПРАВИЛО: дизайн — закон. Мы НЕ натягиваем новый дизайн на старые
компоненты. Каждый экран пересобирается с нуля по моку, из старого кода
переносится только логика (stores, api, hooks, роутинг). Старый компонент и его
CSS удаляются сразу после замены. Никаких хардкод-цветов — только var(--pr-*).

Начни с фундамента (tokens.css + шрифт Unbounded + примитивы PrCard/PrButton/
PrRing/PrSpark/BNav в src/components/ui/), затем иди по экранам в порядке из
README. После каждого экрана: скриншот в обеих темах, сверка с моком, удаление
старого кода, коммит.

Работай инкрементально: один экран = один PR/коммит. Если данные из API не
влезают в макет — решай в языке дизайна (ellipsis, скелетон, эмпти-стейт), не
возвращайся к старой вёрстке. Вопросы по дизайну решай по мокам, они канон.
