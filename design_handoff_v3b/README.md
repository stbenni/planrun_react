## ГЛАВНАЯ ДИРЕКТИВА — прочти перед любым кодом

**Этот дизайн — закон. Функционал навешивается на него, а не наоборот.**

Запрещено:
- ❌ «Подкрашивать» старые компоненты под новый стиль (менять CSS у существующих `DashboardV3`, `CoachWorkspace` и т.д.)
- ❌ Переиспользовать старую вёрстку/разметку «потому что она уже работает»
- ❌ Смешивать: новый экран из старых виджетов или старый экран с новыми кнопками
- ❌ Оставлять старые CSS-файлы подключёнными «на всякий случай»

Обязательно:
- ✅ Каждый экран **пересобирается с нуля** по моку из `mocks/` — мок и есть вёрстка (инлайн-стили в JSX читаются как спека: отступы, размеры, цвета, радиусы)
- ✅ Из старого кода переносится **только логика**: stores (Zustand), api/, hooks/, роутинг, бизнес-правила
- ✅ Старые компоненты и их CSS **удаляются** сразу после замены экрана — в репо не должно остаться двух версий одного экрана
- ✅ Все цвета/радиусы/тени — только через `var(--pr-*)` из `tokens.css`. Хардкод hex в JSX — баг.

Если реальные данные не влезают в макет (длинные имена, пустые состояния) — решение принимается в языке дизайна (ellipsis, скелетоны, эмпти-стейты в стиле карточек), а не возвратом к старой вёрстке.

## Состав пакета
| Файл | Что это |
|---|---|
| `prototype.html` | **Кликабельный прототип всего сайта** — открой в браузере: мобилка/десктоп, роли бегун/тренер, рабочие переходы. Это референс поведения. |
| `mocks/*.jsx` | **Исходники всех экранов** — канонічная вёрстка. `r3-b.jsx` = дашборды + примитивы дизайн-системы, остальные файлы по доменам (см. SCREENS.md). |
| `tokens.css` | CSS-переменные обеих тем + рецепты `.pr-card`, `.pr-btn-primary`, `.pr-chip`, `.pr-label`. Тёмная тема: `html[data-theme="dark"]`. |
| `SCREENS.md` | Маппинг: каждый мок ↔ файлы текущего кода ↔ что меняется. Плюс правила навигации. |

## Порядок работ
1. **Фундамент.** Подключить `tokens.css` (после `sports-colors.css`; по мере миграции старые токены отмирают). Добавить шрифт Unbounded (Google Fonts, 500/600/700, кириллица). Jost остаётся, Montserrat удалить.
2. **Примитивы** в `src/components/ui/`: `PrCard`, `PrButton`, `PrChip`, `PrLabel`, `PrRing`, `PrSpark`, `PrToggle`, `PrLiveDot`, `PrField`, `PrSheet` (bottom-sheet), `BNav` (таб-бар 5 вкладок). Референс-реализации — в `mocks/r3-shared.jsx` (R3Ring, R3Spark, R3Icon) и `mocks/r3-b.jsx` (bCard, bLabel, BNav и пр.). Иконки — из `R3Icon` (stroke 1.8–1.9, скруглённые).
3. **Экраны по порядку** (каждый: пересобрать по моку → подключить логику → удалить старый компонент + CSS):
   онбординг → дашборд бегуна → таб-бар и страница пользователя → календарь → чат → статистика → CoachWorkspace → настройки/уведомления → тренеры и профили → лендинг/auth → инструменты тренера → админка.
4. **Зачистка.** Грепнуть остатки: старые CSS-классы (`dash-v3__`, `cw-`, и т.д.), хардкод-цвета `#FC4C02|#FF4D00`, неиспользуемые файлы. Всё в мусор.

## Навигация (критично)
- Таб-бар бегуна: **Главная / План / Чат / Прогресс / Профиль** (5-я вкладка — страница пользователя, drawer-меню НЕ существует).
- Страница пользователя: шапка аватар+имя+чип режима+**шестерёнка → настройки**; цель, рекорды, лента тренировок, «Мои тренеры».
- Каталог тренеров: из профиля («Мои тренеры»), из режим-чипа на главной («Живой тренер»), с лендинга.
- Полная карта переходов — экран «Карта переходов» в prototype.html (выпадающий список → Карта).

## Definition of Done (по каждому экрану)
- Пиксельно соответствует моку в обеих темах (light/dark — переключатель в prototype.html для сравнения).
- Ни одного хардкод-цвета; только `var(--pr-*)`.
- Старый компонент экрана и его CSS удалены из репо.
- Реальные данные: длинные строки обрезаются, пустые состояния оформлены, загрузка — скелетоны на `--pr-card`.
- Анимации: кольца stroke-dashoffset 1.1s cubic-bezier(.22,1,.36,1); hover карточек translateY(-2px); уважать `prefers-reduced-motion`.

## Ключевые правила языка (шпаргалка)
- Один hero-элемент на экран; остальное — поддержка. Не строить стопки одинаковых карточек.
- Градиент `--pr-grad` — только primary CTA, активные сегменты, кольца, акцент-бордеры.
- Кольца = прогресс к пределу; спарклайны = тренды; сегментные шкалы = зоны.
- Live-точка (pulse) — только у реально живых данных (синк, tool-call, онлайн).
- Labels: 600 / 9–11px / letter-spacing .14em / uppercase / `--pr-sub`. Цифры — Unbounded, текст — Jost.
- Статусы: выполнено `--pr-good` · сегодня/активно `--pr-accent` · пропуск/риск `--pr-bad` · план `--pr-track`.
