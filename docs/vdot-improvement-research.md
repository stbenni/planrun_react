# Исследование: как сделать VDOT более правдоподобным по реальным данным

## Текущая ситуация

- **last_race**: результат забега/контрольной в профиле — может быть устаревшим (месяцы/годы).
- **best_result**: одна тренировка с максимальным VDOT — может быть выброс (короткий спринт, GPS-ошибка, идеальные условия).
- Игнорируется: давность, тип нагрузки, множество тренировок.

## Исследование (источники)

### Jack Daniels (coacheseducation.com, fellrnr.com)

- VDOT рассчитан под **результат гонки**, а не тренировки.
- Оптимальный период: **4–6 недель**.
- Лучшие дистанции для оценки: **2 мили – 10 км** (3–10 км).
- Более длинные (15–25 км) лучше предсказывают марафон.
- Тренировки: тренеры могут оценивать по повторяющимся сессиям, если нет перетренированности.

### Strava Performance Predictions

- **≥20 пробежек** за последние 24 недели.
- ML-модель, **100+ атрибутов**, история и топ-результаты.
- Каждая дистанция считается **отдельно** (5K, 10K, half, marathon).
- Обновление после каждой загрузки или раз в 3 дня без загрузок.

### V.O2 / VDOT tracking

- VDOT должен **меняться** и отражать текущую форму.
- Если не менялся 6 недель — стоит пересмотреть нагрузку и восстановление.
- **Простой 6+ дней** — возможное снижение VDOT.
- Обновление каждые **4–6 недель** по результату гонки/теста.

### Treadmill Buddy / VDOT Calculator

- Три источника: гонка, **тренировочные данные** (темп + пульс), лабораторный тест.
- Тренировочные данные: учитываются **темп и пульс** за несколько недель.

## Рекомендации для PlanRun

### 1. Окно давности (recency)

- Учитывать только тренировки за последние **8–12 недель**.
- Один «лучший» результат за 2 года хуже, чем стабильный темп за последний месяц.

### 2. Фильтр по дистанции

- **2–25 км** — более надёжный диапазон для VDOT.
- Отсекать: 1 км (спринт), >42 км (ультра, другая нагрузка).
- 3–10 км — оптимально для VDOT (Jack Daniels).

### 3. Взвешенное среднее по нескольким результатам

- Вместо одного «лучшего» — взять несколько лучших (например, 5–10).
- Взвешивание по давности: `weight = 0.85^(weeks_ago)`.
- Итог: взвешенное среднее или медиана.
- Снижает влияние выбросов (GPS, короткий спринт).

### 4. Тип тренировки (если есть)

- **race/control** — максимальная нагрузка, вес 1.0.
- **tempo** — порог, ~85–90% VDOT → корректировка: `vdot_estimate = vdot / 0.88`.
- **interval** — сложнее, смешанный темп.
- **long/easy** — обычно ниже, недооценка VDOT.

### 5. Устаревший last_race

- Если `last_race_date` > 8–12 недель — можно «смешивать» с тренировками.
- Варианты: 70% last_race + 30% best_recent, или брать максимум из двух.

### 6. Минимальный объём данных

- Если тренировок < 5 — использовать текущую логику (best_result).
- Если ≥ 5 — переходить на взвешенное среднее.

## Реализация (поэтапно)

**Фаза 1 (без сложности):**
- Окно давности: 12 недель.
- Дистанция: 2–25 км.
- Взвешенное среднее по топ-5 VDOT с recency decay.

**Фаза 2 (если есть plan_type):**
- Учитывать тип тренировки (plan_type из training_plan_days).
- Корректировка для tempo.

**Фаза 3 (опционально):**
- Отдельные предсказания по дистанциям (как Strava).
- Требует больше данных и логики.
