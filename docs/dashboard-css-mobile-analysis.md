# Анализ адаптивности CSS дашборда для мобильных устройств

## Текущие breakpoints

| Breakpoint | Назначение |
|------------|------------|
| **max-width: 720px** | Ряд из двух блоков → одна колонка; паддинг у быстрых метрик |
| **max-width: 768px** | `.dashboard-top-section` (2 колонки) → одна колонка |
| **max-width: 640px** | Уменьшение паддинга контейнера, приветствие, метрики |
| **min-width: 900px** | Сетка статистики: 4 колонки (иначе 2×2) |
| **min-width: 1024px** | Отступ сверху под TopHeader (88px) |

---

## Что уже хорошо

1. **Контейнер**
   - `max-width: 1400px` + `margin: 0 auto` — не фиксированная ширина.
   - Отступы через переменные `var(--space-5)`, `var(--space-6)`.

2. **Сетки и раскладки**
   - `.dashboard-row-two`: `grid-template-columns: 1fr 1fr` → на 720px переключается в `1fr` (одна колонка).
   - `.dashboard-top-section`: на 768px переходит в одну колонку.
   - Используются `min-width: 0`, `flex: 1` — контент не «раздувает» сетку на узких экранах.

3. **Мобильный блок (640px)**
   - У контейнера: `padding: 16px`.
   - Приветствие: `font-size: var(--text-2xl)`.
   - Быстрые метрики: меньшие отступы, иконки, шрифты.

4. **Скролл и касания**
   - `-webkit-overflow-scrolling: touch` для плавного скролла на iOS.
   - `padding-bottom: 100px` под нижнюю навигацию.

5. **Кастомные блоки**
   - Окно кастомайзера: `max-width: 400px`, `width: 100%`, `max-height: 90vh` — не вылезает за экран.

---

## Проблемы и риски на мобильных

### 1. Разные пороги для «мобилки»
- 720px и 768px — разный момент перехода в одну колонку.
- 640px — единственное место, где уменьшается общий паддинг и часть типографики.
- На ширинах 320–640px паддинг по умолчанию остаётся `var(--space-5)` / `var(--space-6)` (20px / 24px), что на 320px забирает заметную долю ширины.

**Рекомендация:** ввести один основной мобильный breakpoint (например 640px или 480px) и в нём согласовать: паддинги, размеры заголовков, отступы в карточках.

### 2. Тёмная тема на узких экранах
- В тёмной теме к паддингам добавляется по 40px с каждой стороны.
- На ширине 360px полезная ширина: 360 − 2×(20+40) = 240px — очень мало.

**Рекомендация:** в медиа (max-width: 640px) для `[data-theme="dark"] .dashboard` уменьшить добавку (например до 12–16px или убрать).

### 3. Шапка дашборда
- `.dashboard-header-row`: `display: flex`, `justify-content: space-between`, без `flex-wrap`.
- При узком экране приветствие и кнопка «Настроить» могут сильно сжиматься или наезжать друг на друга.

**Рекомендация:** на мобильном (например ≤640px) сделать перенос: `flex-wrap: wrap` и/или уменьшить размер приветствия (уже есть на 640px) и кнопки.

### 4. Фиксированная высота карточек в ряду
- В ряду из двух блоков: `height: 230px`, `min-height: 230px`.
- На очень маленькой высоте экрана (например 568px с учётом шапки и навбара) блоки занимают много места по вертикали.

**Рекомендация:** в мобильном breakpoint можно задать меньшую высоту (например 200px) или `min-height` с сохранением `height: auto` при одной колонке.

### 5. Кастомайзер на маленьком экране
- `.dashboard-customizer-row-double`: `grid-template-columns: 1fr 1fr` без изменений на узкой ширине.
- В overlay 400px две колонки ещё ок, но при ширине viewport 320px и padding 20px каждая колонка ~130px — тесно для подписей.

**Рекомендация:** при max-width: 400px (или 480px) для строк кастомайзера переключать в одну колонку.

### 6. Размеры касаний (touch targets)
- Рекомендуемый минимум ~44×44px.
- Сейчас: кнопка закрытия 36×36, кнопки периода 8px 14px padding, стрелка раскрытия — небольшая область.
- Часть кнопок может быть маловата для удобного нажатия пальцем.

**Рекомендация:** в мобильном медиа увеличить `min-height`/`padding` у основных кликабельных элементов до ~44px по высоте.

### 7. Крупные фиксированные отступы
- Много блоков с `margin-bottom: 40px`, `padding: 24px`, `gap: 16px`.
- На 320px по вертикали это съедает много места.

**Рекомендация:** в мобильном медиа уменьшить до 16–24px (margin-bottom, padding секций).

### 8. Нет отдельного очень узкого breakpoint
- Для ширин 320–400px (маленькие смартфоны) все правила заданы общим блоком 640px.
- Шрифты и отступы можно дополнительно уменьшить для 360px и ниже (опционально).

---

## Итог

- База адаптивности есть: колонки перестраиваются, часть мобильных правок на 640px и 720px задана.
- Главные точки роста: **единый мобильный breakpoint**, **паддинги и отступы на узких экранах**, **тёмная тема на мобильных**, **шапка и кастомайзер**, при необходимости — **размеры touch-элементов** и **высота карточек в ряду**.
