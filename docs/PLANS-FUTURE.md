## 6. Сложность тренировки — погода и поверхность

**Идея:** Одна и та же тренировка (например, 10 км easy) может быть разной по нагрузке: бег по снегу зимой vs асфальт летом. Нужна стратегия учёта условий для корректной оценки нагрузки и сравнения результатов.

### Исследование (2025)

**Погода:**
- Оптимум: 7–15°C. Выше — замедление ~1.7–4.5% на каждые 5°C (Runners World, Outside, PLOS One). Холод сам по себе влияет меньше, чем поверхность (сугробы vs чистый асфальт).
- Жара + влажность: при 27°C и 90% влажности темп падает сильнее, чем при той же температуре и 30% влажности (Strava 2023).
- Ветер: встречный 16 км/ч добавляет ~8–15 сек/км.
- Дождь/снег: скользко, одежда тяжелеет, видимость.

**Поверхность (важнее температуры):**
- Асфальт — база (1.0). Холодный асфальт зимой (чистый) — почти та же нагрузка, что летом.
- Сугробы — сильно тяжелее: нестабильная опора, проваливание, энергозатраты выше. Замедление может превышать 1 мин/км при том же усилии. Формулы с глубиной снега (ScienceDirect: η = f(z)).
- Утоптанный снег — легче сугробов, но тяжелее асфальта.
- Трейл — +15% к «эквивалентному» темпу.
- Грязь — аналогично снегу.

**Подходы (Strava, Garmin, Stryd):**
- Grade Adjusted Pace (GAP) — учёт рельефа.
- Running Power — учитывает ветер (Stryd).
- Погоду пока массово не учитывают; в основном — рельеф.

### Предлагаемая реализация

| Этап | Задача | Описание |
|------|--------|----------|
| 1 | Погода | OpenWeatherMap History API: lat, lon, timestamp → temp, humidity, wind, rain/snow. Координаты из GPS трека (GPX/TCX, Strava) — start_lat, start_lon. Кеш по (lat, lon, date). |
| 2 | Множитель сложности | Формула: `difficulty = f(temp, humidity, wind, surface)`. База — асфальт, 10°C, без ветра. Коэффициенты из исследований. |
| 3 | Тип поверхности | Поле в save_result: road, trail, snow_packed (утоптанный), snow_deep (сугробы), mud, treadmill. Сугробы — отдельный уровень сложности, не путать с «просто холод». |
| 4 | Эквивалентный темп/нагрузка | При отображении результата: «5:30/км (эквивалент ~5:00 на асфальте)» или «нагрузка 1.15× от базовой». |
| 5 | Интеграция в VDOT/статистику | Учёт difficulty при расчёте формы и сравнении тренировок. |

### GPS → местоположение → погода

**Идея:** По треку тренировки (GPX/TCX, Strava) определять координаты и получать погоду на момент тренировки — без ручного ввода.

**Что есть:**
- GpxTcxParser извлекает lat/lon из каждой точки, но не сохраняет — только distance, elevation.
- workout_timeline хранит timestamp, altitude, pace — без lat/lon.
- Таблицы workouts, workout_log — без полей start_lat, start_lon.

**Что нужно:**
1. При импорте GPX/TCX и синке Strava — сохранять start_lat, start_lon (первая точка или центроид маршрута). Миграция: добавить колонки в workouts; для workout_log — опционально (ручные записи без GPS).
2. OpenWeatherMap History API: `?lat={lat}&lon={lon}&dt={unix_timestamp}` — историческая погода на момент старта тренировки (temp, humidity, wind, rain/snow).
3. Кеш по (lat, lon, date) — один запрос на день/локацию.
4. Fallback для ручных записей: город пользователя (профиль) или запрос погоды по умолчанию.

**Итог:** Точные условия (температура, ветер, осадки) в момент тренировки — автоматически для импортированных треков.

**Файлы:** GpxTcxParser (вернуть/пробросить points или centroid), WorkoutService.importWorkouts, StravaProvider — сохранять lat/lon; миграция workouts; WeatherDifficultyService + OpenWeatherMap History API.
